# 医院管理系统 7.0

这是一个基于 C/C++ 与 Visual Studio 的医院信息管理系统课程设计项目。系统围绕医院门诊业务展开，覆盖患者挂号、预约报到、签到排队、医生接诊、处方开具、收费结算、医保报销、药房发药、住院管理、出院结算、管理员统计和医生管理等完整流程。

> 说明：工程文件名仍沿用 `医疗管理系统6.0`，但源码已经包含 7.0 功能，例如医保类型、药品甲乙丙类别、患者签到队列、预约报到、病房可视化等。

## 一、快速登录入口

运行程序后进入主菜单：

```text

1. 患者

3. 药房/收费管理人员
4. 医院管理人员 (数据分析)
5. 推进时间一天
6. 重置/初始化系统数据
7. 手动保存数据
```

## 二、科室编号

| 编号 | 科室 | 主要用途 |
| --- | --- | --- |
| `1` | 急诊科 | 急诊现场挂号、急救用药、紧急 ICU 转入 |
| `2` | 内科 | 常规内科疾病、慢病、消化、血压、血糖等 |
| `3` | 外科 | 外伤、创口、外科处理及部分住院场景 |
| `4` | 儿科 | 未成年患者常用入口，儿科药品和儿童处方 |
| `5` | 妇科 | 妇科专属挂号、妇科用药 |

挂号限制：

- 未满 18 岁患者只能挂急诊科或儿科。
- 成年患者不能挂儿科。
- 男性患者不能挂妇科。
- 急诊科不支持预约挂号，只支持现场挂号。
- 患者已有未完成就诊流程时，不能重复挂号。

## 四、药品、类别与科室关联

系统内置 20 种药品。每种药品有三类信息：

| 字段 | 含义 |
| --- | --- |
| 通用名 | 医学或标准名称，例如“乙酰水杨酸” |
| 商品名 | 市售名称，例如“拜阿司匹林” |
| 别名 | 俗名或快捷输入名，例如“阿司匹林” |
| 药品类别 | 医保结算用，分为甲类、乙类、丙类 |
| 适用科室 | 决定哪些科室医生可以直接开具该药 |

药品类别与医保规则：

| 类别 | 系统含义 | 居民医保 | 职工医保 |
| --- | --- | --- | --- |
| 甲类 | 基础报销药品 | 药费报销 70% | 药费报销 80% |
| 乙类 | 先自付 5%，剩余部分再按比例报销 | 剩余 95% 报销 70% | 剩余 95% 报销 80% |
| 丙类 | 自费药品 | 不报销 | 不报销 |

检查费、诊疗费、床位费的医保规则：

| 医保类型 | 其他费用报销比例 |
| --- | --- |
| 无医保 | 0% |
| 居民医保 | 50% |
| 职工医保 | 70% |

科室权限规则：

- `全科通用` 表示所有科室医生都可直接开具。
- 非全科药品只允许表中列出的科室直接开具。
- 医生开具非本科室常用药时，系统会提示越权风险，需要确认后才会记录。

- 初始库存为每种药品 1000。
- 医生开具处方后，患者需要先缴费。
- 药房发药时才会真实扣减库存。

## 五、患者端操作指南

主菜单选择 `1. 患者` 进入患者自助终端。患者端不需要账号密码，主要通过患者 ID 或手机号查询本人信息。

| 菜单项 | 功能 | 使用说明 |
| --- | --- | --- |
| `1` | 门诊挂号 | 新患者注册或复查患者挂号，填写姓名、性别、生日、手机号、医保类型，选择科室和医生 |
| `2` | 预约挂号 | 选择科室、医生、未来 7 天日期和上午/下午时段；急诊科不能预约 |
| `3` | 预约管理 | 查看已有预约，也可取消预约 |
| `4` | 患者签到 | 挂号后先签到，医生端按签到顺序叫号 |
| `5` | 查看医嘱 | 看诊后查看处方、检查费、药费、住院建议和缴费状态 |
| `6` | 打印电子发票 | 查看历史缴费发票、医保报销记录和退费凭证 |
| `7` | 自助办理住院 | 医生开具住院建议并完成缴费后，选择普通或 VIP 病房入住 |
| `8` | 自助办理出院 | 结清住院费用后办理出院，释放床位 |
| `9` | 打印挂号单 | 通过患者 ID 或手机号查看挂号信息和诊疗记录 |
| `10` | 自助缴费 | 患者可按 ID 或手机号缴费，并选择是否使用医保 |

建议演示流程：

1. 进入患者端，选择 `1. 门诊挂号`。
2. 注册一名新患者，填写医保类型。
3. 选择内科或外科医生。
4. 返回患者端选择 `4. 患者签到`。
5. 记录系统生成的患者 ID，后续医生接诊、缴费、发药都需要用到。

## 六、收费与药房操作指南

主菜单选择 `3. 药房/收费管理人员`，登录账号为 `admin`，密码为 `123`。

| 菜单项 | 功能 | 使用说明 |
| --- | --- | --- |
| `1` | 办理患者缴费 | 输入患者 ID，系统展示诊疗费、检查费、药费、床位费，并结算费用 |
| `2` | 药房核对发药 | 输入患者 ID，校验是否已缴费，发药后扣减药品库存 |
| `3` | 查看全院药品及库存明细 | 查看药品 ID、通用名、进价、售价、库存、适用科室、累计售出 |
| `4` | 药房补货采购进货 | 按药名补货；药品不存在时可新建药品档案 |
| `0` | 退出登录 | 返回系统主菜单 |

收费规则：

- 患者看诊后产生诊疗费、检查费、药费。
- 需要住院时会产生床位费。
- 患者未缴清费用时，药房拒绝发药，住院患者也不能出院。
- 患者有医保时，可在缴费时选择是否使用医保报销。

新建药品档案需要填写：

- 通用名、商品名、别名。
- 进货单价、对患者零售单价、首批库存。
- 是否全科通用；如果不是，则逐个设置急诊科、内科、外科、儿科、妇科是否可用。
- 药品类别：`0` 甲类、`1` 乙类、`2` 丙类。

## 七、医院管理员操作指南

主菜单选择 `4. 医院管理人员`，管理员工号为 `9`，密码为 `a`。

| 菜单项 | 功能 | 使用说明 |
| --- | --- | --- |
| `1` | 患者信息查询 | 查看全部患者，或按患者 ID、姓名、科室、就诊状态查询，也可导出患者信息 |
| `2` | 科室信息查询 | 查看各科室接诊人数、营收和在职医生情况 |
| `3` | 住院部入住情况管理 | 查看病房入住情况、手动办理出院、显示病房占用格子图 |
| `4` | 资金流动查询 | 统计诊疗费、检查费、药费、住院费等收入，并支持导出 |
| `5` | 医生管理 | 查看在职医生、新增医生、开除医生、查看离职医生 |
| `6` | 全院医生排班与待诊看板 | 查看医生待诊人数和排班状态 |
| `0` | 退出登录 | 退出管理员后台，并保存系统数据 |

医生管理规则：

- 新增医生时需要填写姓名、科室、职级、挂号费和初始密码。
- 开除医生并不会物理删除数据，而是标记为离职，方便保留历史诊疗记录。
- 开除医生前，系统会检查该医生是否还有待诊患者或未来预约；如果存在，会拒绝开除。
- 离职医生不能再登录医生端。

## 八、住院管理说明

住院流程由医生端、患者端和收费端共同完成：

1. 医生看诊时判断患者是否需要住院。
2. 患者先完成门诊费用缴费。
3. 患者端选择 `7. 自助办理住院`。
4. 系统根据科室和病房类型分配床位。
5. 入住后会产生床位费用。
6. 出院前必须先结清床位费。
7. 患者端选择 `8. 自助办理出院` 后释放床位。

病房类型：

| 类型 | 说明 |
| --- | --- |
| 普通病房 | 常规住院床位 |
| VIP 病房 | 高费用住院床位 |
| ICU | 急诊医生可在紧急情况下直接转入 |

## 九、数据保存与重置

系统使用 `his_data.dat` 保存医生、患者、药品、病房、预约、病历和模拟日期。

| 操作 | 说明 |
| --- | --- |
| 首次运行 | 如果找不到 `his_data.dat`，系统会初始化医生、药品、病房并生成演示患者 |
| 手动保存 | 主菜单选择 `7. 手动保存数据` |
| 自动保存 | 退出系统时自动保存 |
| 重置数据 | 主菜单选择 `6. 重置/初始化系统数据`，会清空患者、预约、病历，并恢复基础数据 |
| 恢复首次运行效果 | 删除 `his_data.dat` 后重新启动程序 |

系统默认模拟日期为 `2026-03-09`，主菜单选择 `5. 推进时间一天` 可推进日期，便于演示预约、住院天数和时间流转。

## 十、运行方式

推荐使用 Windows + Visual Studio。

1. 安装 Visual Studio，并勾选“使用 C++ 的桌面开发”。
2. 打开 `医疗管理系统6.0.slnx` 或 `医疗管理系统6.0.vcxproj`。
3. 选择 `Debug | x64` 或 `Release | x64`。
4. 点击“本地 Windows 调试器”运行。

也可以使用 Visual Studio Developer PowerShell 构建：

```powershell
msbuild .\医疗管理系统6.0.vcxproj /p:Configuration=Debug /p:Platform=x64
```

## 十二、课堂展示建议

一条完整演示链路如下：

1. 患者端注册新患者，选择医保类型并现场挂号。
2. 患者完成签到。
3. 医生端使用对应科室医生账号登录。
4. 医生按签到顺序叫号，查看病历，开具处方、检查费和医嘱。
5. 患者端或收费端完成缴费，并演示医保报销。
6. 药房端核对处方并发药，展示库存扣减。
7. 对需要住院的患者演示住院办理、床位分配和出院结算。
8. 管理员后台查看患者状态、科室统计、资金流动、医生管理和病房占用图。

## 十三、项目亮点

| 模块 | 亮点 |
| --- | --- |
| 患者端 | 支持新患者注册、复查挂号、预约、签到、缴费、发票、住院和出院 |
| 医生端 | 支持医生账号登录、签到叫号、历史病历、处方录入、住院建议和 ICU 转入 |
| 药房收费 | 支持医保计算、费用结算、发票记录、库存扣减、缺药退费和药品采购 |
| 管理员后台 | 支持患者查询、科室统计、资金分析、医生管理、排班看板和病房格子图 |
| 数据结构 | 使用链表管理医生、患者、药品、病房、预约和病历 |
| 数据持久化 | 使用二进制文件 `his_data.dat` 保存系统运行状态 |

## 十四、项目性质

本项目用于课程设计、课堂展示和 C/C++ 数据结构实践，重点展示控制台交互、链表管理、业务流程建模、文件持久化和基础权限控制，不用于真实医疗场景。
